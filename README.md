# poly_trade

This repo is for **researching and backtesting BTC Up/Down 5-minute Polymarket data**.

The current main workflow is focused on one concrete question:

> Given the **first 2 minutes** of BTC price action and order-book information, what can best predict the **remaining 3 minutes / final 5-minute outcome**?

## Data source

The source data currently comes from:

- `yt-feng/poly/data/runs/24869603988_attempt1`

The research workflow pulls that source repo directly inside GitHub Actions, so you do not need to prepare local files.

## Repo layout

- `analysis/build_run_24869603988_dataset.py` — clean the raw quote chunks, build first-2-minute features, test several common strategies, and generate charts + report
- `.github/workflows/run-24869603988-analysis.yml` — runs the research and commits outputs back to this repo
- `data/cleaned/run_24869603988_attempt1/` — cleaned quote-level data generated by the workflow
- `data/features/run_24869603988_attempt1/` — market-level first-2-minute feature table generated by the workflow
- `reports/run_24869603988_attempt1_predictive/` — report, figures, strategy tables, and summary JSON generated by the workflow

## Research logic

The current report is designed to answer questions like:

- If BTC already moved **+30 USD** in the first 2 minutes, does that make an Up finish more likely or less likely?
- Is **momentum** better or is **mean reversion** better?
- Can simple ML models beat hand-written threshold rules after paying an entry price and a fixed fee?

The workflow currently compares:

- majority-class baseline
- momentum threshold rule on first-2-minute BTC move
- mean-reversion threshold rule on first-2-minute BTC move
- logistic regression
- random forest

## How to run

Open the **Actions** tab and run:

- `Build and commit first-2-minute predictive research`

After it finishes, outputs should be written back into this repo under `data/` and `reports/`.

## What to read after a run

The most useful file is:

- `reports/run_24869603988_attempt1_predictive/report.md`

That report should include:

- data summary
- strategy comparison table
- threshold search results
- chart set
- trade log sample
- generated figures stored in the repo
